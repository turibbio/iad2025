# Linee guida per contribuire

Grazie per l'interesse a contribuire a questo progetto! Per favore segui il workflow tramite fork: non aprire pull request direttamente sul repository principale.

## Workflow consigliato (passo-passo)

1. Fai fork del repository nel tuo account GitHub.
2. Clona il tuo fork in locale:

```bash
git clone https://github.com/<tuo-username>/iad2025.git
cd iad2025
```

3. Crea una branch con nome descrittivo:

```bash
git checkout -b feature/nome-descrittivo
```

4. Fai i tuoi cambiamenti, mantieni i commit piccoli e con messaggi chiari.
5. Push della branch sul tuo fork:

```bash
git push origin feature/nome-descrittivo
```

6. Apri una Pull Request (PR) dal tuo fork verso il branch `main` del repository principale.

## Cosa includere nella PR

- Riferimento a eventuale issue (`Fixes #numero` o `Related to #numero`).
- Descrizione breve e chiara del cambiamento.
- Istruzioni per testare le modifiche (se applicabili).
- Screenshot o esempi, se utili.

## Stile dei commit

- Usa messaggi in inglese o italiano chiari e descrittivi.
- Preferisci commit atomic e ben suddivisi (una cosa per commit).
